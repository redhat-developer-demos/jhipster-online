Create OpenShift Application

Generate a JHipster app tuned for OpenShift (MariaDB, Tekton, Helm chart, optional Argo CD) and deploy from your Git repository.

  • Step 1: Open the OpenShift generator and fill the application form
  • 2 Step 2: Optionally add extra Kubernetes / Helm YAML (HPA, routes, operators)
  • 3 Step 3: Push to Git, then deploy to your namespace (Fabric8 or Argo CD)

Create MCP Server

Scaffold a Model Context Protocol server from templates (Spring AI MCP, Quarkus, .NET, or Python), optionally generate CRUD-style MCP tools from your JDL with AI, preview files, then download a ZIP.

  • 1 Step 1: Choose framework (Spring Boot, Quarkus, .NET, Python)
  • 2 Step 2: Optionally paste JDL entities to auto-generate MCP tools with AI
  • 3 Step 3: Preview and customize generated code in the visual editor
  • 4 Step 4: Download ZIP (requires mcp-worker sidecar when running the stack)

Deployed Applications

List and remove Helm-managed releases that JHipster Online deployed to your OpenShift project (label-based uninstall).

  • Step 1: Choose your target namespace and refresh the list of deployed applications
  • Step 2: Delete an application when you no longer need it in the cluster
